# 给未来的自己 - 复利工程

## 你会失忆

每次被唤醒，你都是一张白纸。你不会记得：
- 上次讨论了什么
- 为什么做了那个决定
- 踩过的坑、犯过的错
- 代码审查中学到的东西

这很正常，别慌。

## 核心理念

**每一次纠正、每一次优化，都应该记录下来。** 不是只为了这次任务完成，而是为了下一次不再犯同样的错误。这就是"复利工程"。

## 运行步骤

⚠️ **复利是自动行为，不是可选流程。每次任务完成后，不要等用户提醒，直接执行。**

### 1. 回顾

任务完成时，问自己：有没有被纠正过？有没有发现规范 gap？只有真正学到了新东西才记录，不要为了记录而记录。

### 2. 写入

根据内容类型选择位置：

| 内容类型 | 位置 | 写法 |
| --- | --- | --- |
| 常见代码错误 | `docs/common-mistakes.md` | ⚠️ 标题 + 一句话描述 + `file:xxx:line` 代码坐标引用 |
| 项目知识、决策记录 | `docs/` 目录 | 具体经验和详细指导 |
| 代码风格、坑位 | `.catpaw/rules/codejoy/code-style.md` | 给 AI 看的规则（自动生成，不要手动改） |

### 3. 告知用户

```
📝 已复盘，经验已记录：
- docs/common-mistakes.md 新增：[标题]
```

**核心原则**：复盘是静默的、自动的。做完直接告诉结果，不要问"要不要复盘"。

## 写作标准

1. **简短** - 用 `file:xxx:line` 引用代码，不要大段复制
2. **直接** - 说人话，不要废话
3. **可执行** - 不只说"不要这样做"，要说"应该这样做"

### ❌ 太笼统

```markdown
配置要注意大小写
```

### ✅ 具体

```markdown
## YAML frontmatter 双引号必须转义

⚠️ YAML frontmatter 中的双引号必须转义

❌ 未转义导致 YAML 解析失败
✅ 用反斜杠转义内部双引号

参考：`file:src/rules/generator/ruleGenerator.ts:145`
```

## AGENTS.md 与 docs/ 的关系

**渐进式披露**：AGENTS.md 是导航中心（摘要 + 索引），docs/ 是详细内容。

- ✅ AGENTS.md 中说"参见 docs/xxx"——指路，不重复
- ❌ 不要在 AGENTS.md 中复述 docs/ 的内容——浪费上下文
- ✅ docs/ 中要有"何时用"的标注——"CodeReview 时检查本文"
- ✅ docs/ 中要有具体例子——指向代码位置（`file:xxx:line`），不贴完整代码块
- ✅ AGENTS.md 提供常见反模式/错误清单（标题列表），docs/ 补充 know-how（为什么错、怎么改）
- ❌ 不要在 AGENTS.md 中展开讲解，那是 docs/ 的职责
